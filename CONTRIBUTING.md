# Contributing to Automata

We welcome contributions to Automata! As a highly systemic application engine, we maintain strict code quality and architectural standards.

## Development Process
1. Fork the repository.
2. Create your feature branch (`git checkout -b feature/amazing-feature`).
3. Commit your changes (`git commit -m 'feat: Add amazing feature'`).
4. Ensure all tests pass (`make test`).
5. Push to the branch (`git push origin feature/amazing-feature`).
6. Open a Pull Request.

## Code Style
We enforce strict TypeScript typing and linting. Refer to the `.eslintrc.json` for details.
